Local cochain map for nodal H¹ DOFs on a given geometry.

The Cochain class encodes how degrees of freedom from boundary entities (vertices, edges, faces) are injected into the DOF array of a polytope. This is essential for building conforming H1 spaces where shared entities have shared DOFs.

For example, when numbering DOFs on a triangle, the Cochain::map method injects vertex and edge DOFs into the triangle's local DOF array, respecting the reference element's nodal ordering.

See also

getClosure() which uses Cochain::map during DOF numbering

This class encodes, at compile time, how the nodal degrees of freedom living on the boundary entities of a reference polytope G are injected into the nodal DOFs of G itself.

Concretely:

  • G is the codomain polytope (Segment, Triangle, Quadrilateral, Tetrahedron, or Wedge),
  • Domain is a container of DOF indices on the boundary entity,
  • Codomain is a container of DOF indices on the whole polytope G.

The method Cochain::map writes appropriate entries of codomain from the entries of domain, respecting:

  • the canonical reference-geometry vertex ordering in Polytope::Traits,
  • the edge / face orientation conventions in Connectivity::getSubPolytopes,
  • the nodal ordering used in:
    • GLL<K> / GLL01<K> on segments,
    • FeketeTriangle<K> on triangles,
    • FeketeTetrahedron<K> on tetrahedra,
    • tensor-product GLL nodes on quadrilaterals,
    • tensor-product (triangle × GLL in z) on wedges.

From the algebraic viewpoint, this realizes the local cochain map associated with the trace operator on the boundary of G, written in the nodal basis.

Local DOF counts per reference geometry

These constants describe how many nodal DOFs are attached to each reference polytope for the degree K H¹ element.

Number of nodal DOFs on a reference point (single vertex).

Number of nodal DOFs on a reference segment (K+1 GLL nodes).
static size_t TriangleCount constexpr

static size_t TetrahedronCount constexpr

static size_t QuadrilateralCount constexpr
Number of nodal DOFs on a reference quadrilateral ((K+1)×(K+1) GLL grid).

Number of nodal DOFs on a reference hexahedron ((K+1)³ tensor GLL grid).

Injects boundary DOFs into the DOFs of the polytope G.

Parameters
local in Index of the boundary entity of G (vertex, edge, or face, depending on the dimension)
codomain out DOF container on G to be updated along the Local-th boundary entity.
domain in DOF container on the Local-th boundary entity.

This method maps the degrees of freedom associated with a boundary entity of G (vertex, edge, or face) into the degrees of freedom associated with the whole polytope G, for the Local-th boundary entity.

The specific pattern depends on:

  • the codomain geometry G,
  • the Local boundary index (edge/face index),
  • the polynomial degree K and the chosen nodal sets (GLL, Fekete, tensor-product).

The mapping is consistent with:

  • the reference vertex coordinates in Polytope::Traits::getVertex,
  • the edge/face orientations used in Connectivity::getSubPolytopes,
  • the reference nodal ordering on each geometry (getNodes).
