#include <Rodin/Variational/FiniteElement.h>

Rodin::Variational::FiniteElementBase class

Base class for finite elements.

Template parameters
Derived Concrete finite element type implementing the CRTP interface.

Contents

  • Reference
    • Public types
    • Constructors, destructors, conversion operators
    • Public functions

This class provides a minimal CRTP interface shared by all finite elements. Derived classes are expected to implement the following methods:

  • size_t getCount() const
  • auto getBasis(size_t) const
  • auto getLinearForm(size_t) const
  • size_t getOrder() const

The base class stores only the reference geometry and forwards all finite-element-specific queries to the Derived class.

template<class Derived>
Geometry::Polytope::Type Rodin::Variational::FiniteElementBase<Derived>::getGeometry() const constexpr

Returns the reference geometry of the finite element.

Returns Geometry::Polytope::Type representing the reference geometry.

The geometry identifies the reference domain used to define the element (segment, triangle, quadrilateral, etc.). It is typically used to:

  • Select appropriate quadrature rules.
  • Interpret reference coordinates.
  • Build geometry-dependent DOFs and basis functions in Derived classes.

template<class Derived>
size_t Rodin::Variational::FiniteElementBase<Derived>::getCount() const constexpr

Gets the number of degrees of freedom in the finite element.

Returns Total number of local DOFs associated with the element.

This forwards to the Derived class via CRTP: static_cast<const Derived&>(*this).getCount();

For polynomial elements, this is typically the number of interpolation nodes (scalar DOFs) on the reference element.

template<class Derived>
decltype(auto) Rodin::Variational::FiniteElementBase<Derived>::getLinearForm(size_t i) const constexpr

Gets the i-th linear functional (degree of freedom) on the element.

Parameters
i Local DOF index.
Returns Linear functional object representing the i-th degree of freedom.

A linear form represents one element of the DOF set , such as:

  • Point evaluation at a node (for nodal elements),
  • Moments against test polynomials (for non-nodal elements).

Applying the linear form to a function typically produces the coefficient associated with the i-th degree of freedom.

Note

CRTP method: implemented by the Derived class.

template<class Derived>
size_t Rodin::Variational::FiniteElementBase<Derived>::getOrder() const constexpr

Returns a characteristic polynomial order of the finite element.

Returns Polynomial order (total degree or upper bound).

For polynomial-based elements, this is defined as the maximum total polynomial degree of the shape functions on the reference element, or a safe upper bound for it. It is typically used to select appropriate quadrature rules (e.g. rules exact for polynomials up to degree getOrder(), or 2*getOrder() for bilinear forms).

The precise meaning may be refined in the Derived class documentation (e.g. vs elements, tensor-product spaces, etc.).

Note

CRTP method: implemented by the Derived class.
