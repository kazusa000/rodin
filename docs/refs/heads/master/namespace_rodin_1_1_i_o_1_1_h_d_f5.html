Rodin::IO::HDF5 namespace

HDF5 utility namespace providing type aliases, path constants, RAII handle wrappers, and low-level dataset read/write helpers.

Contents

  • Reference
    • Namespaces
    • Classes
    • Typedefs
    • Functions
    • Variables

This namespace contains all HDF5-specific helpers used by the MeshLoader, MeshPrinter, GridFunctionLoader, and GridFunctionPrinter specializations for the HDF5 file format. It wraps the HDF5 C API in a type-safe, RAII-managed interface.

void Rodin::IO::HDF5::writeXDMFMesh(const boost::filesystem::path& filename, const Geometry::MeshBase& mesh)
#include <Rodin/IO/HDF5.h>

Writes a minimal XDMF visualization mesh file.

Parameters
filename in Output HDF5 file path.
mesh in Local mesh to export for visualization.

Creates a new HDF5 file containing only the datasets required for XDMF/ParaView visualization:

  • /Mesh/Geometry/Vertices — vertex coordinates
  • /Mesh/XDMF/Topology — XDMF mixed-topology stream
  • /Mesh/XDMF/TopologySize — length of the topology stream
  • /Mesh/Attributes/{d} — polytope region labels

This function does not write the full canonical Rodin persistence data (connectivity CSR, incidence, state, transformations). Use MeshPrinter<FileFormat::HDF5> for full persistence.

See also

writeXDMFTopology, writeXDMFVertices, writeXDMFRegionAttribute, MeshPrinter<FileFormat::HDF5, Context::Local>

static U64 Rodin::IO::HDF5::NullAttributeMarker constexpr
#include <Rodin/IO/HDF5.h>

Sentinel value used to represent the absence of an attribute in HDF5 attribute datasets.

Polytopes without an assigned attribute are stored as this value in the /Mesh/Attributes/{d} arrays. On load, entries equal to this marker are left unset.
